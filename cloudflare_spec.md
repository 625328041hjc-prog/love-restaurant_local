# Cloudflare 版本部署与功能规格文档

## 1. 架构变更 (从 Supabase 迁移至 Cloudflare)
*   **前端宿主**: Cloudflare Pages (全球 CDN 加速，国内直连更稳定)
*   **后端接口**: Cloudflare Pages Functions (Serverless 函数，自动挂载在 `/api` 路由下)
*   **数据库**: Cloudflare D1 (Serverless SQLite 数据库，已创建 `love_restaurant`)

## 2. 新增功能规格
为了让系统更温馨、更有情侣互动感，我们将实现以下功能：
1.  **UI 全面升级 (温馨风格)**:
    *   主题色调整为暖色调（粉色/橘色点缀）。
    *   卡片增加圆角、阴影，按钮增加柔和的交互动画。
2.  **"不知道吃啥" 盲盒功能**:
    *   在菜单顶部增加一个“盲盒”按钮。
    *   点击后通过随机算法从现有菜品中抽选一道菜，带有翻牌或转盘的动画效果，增加趣味性。
3.  **"热销/推荐" 菜品展示**:
    *   在点餐页面增加一个“今日推荐”或“招牌菜”横幅/滑动列表，优先展示特定的菜品。
4.  **订单评价/留言功能**:
    *   女友在用餐后，可以在订单历史中对这顿饭进行评价（打分 + 留言，比如“太咸了”、“超好吃”）。
    *   后台管理界面可以看到这些评价。

## 3. 改造步骤与开发计划
*   **第一步 (当前阶段)**: 确认本规格文档。
*   **第二步 (代码改造)**: 
    *   移除所有 Supabase 相关的依赖和代码。
    *   在项目根目录新建 `functions/api` 目录，编写连接 D1 数据库的后端接口（获取菜单、下单、获取订单、评价等）。
    *   修改前端代码，调用新的 `/api` 接口，并实现上述盲盒、评价等温馨 UI。
*   **第三步 (本地测试)**: 在当前开发环境中模拟 Cloudflare 环境进行全面测试。
*   **第四步 (上传 GitHub)**: 测试通过后，我会整理好修改过的文件，你需要将它们再次覆盖上传到你刚才建好的 `love-restaurant_local` 仓库中。
*   **第五步 (Cloudflare 部署)**: 带你在 Cloudflare 仪表盘中关联该 GitHub 仓库，绑定 D1 数据库，完成最终的自动化部署。
